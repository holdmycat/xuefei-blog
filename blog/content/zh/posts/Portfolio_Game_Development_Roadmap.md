# Portfolio Game Development Roadmap (8-Week Plan)

**总周期**: 2026年01月08日 - 2026年03月05日 (预计)
**目标**: 完成 "Mini-SoulMaster" (无尽冬日 Like) 竖版 Demo，并上线 TapTap 进行技术与设计验证。

---

## 📅 总体时间轴 (Timeline)

### Phase 1: 工程基建与核心重构 (Week 1-2)
>
> **焦点**: 技术地基、竖版化、热更架构

* **Week 1 (01/08 - 01/14)**: **基建周**
  * [ ] **竖屏重构**: 修改 Camera FOV，重构 Battle UI 为竖版布局。
  * [ ] **HybridCLR**: 接入 HybridCLR，划分 AOT/Hotfix 程序集，跑通代码热更流程。
  * [ ] **Addressables/CDN**: 建立资源管理规范，配置模拟 CDN 加载。
* **Week 2 (01/15 - 01/21)**: **战斗闭环周**
  * [ ] **战斗逻辑**: 实现“能量-大招”、“前后排仇恨”逻辑。
  * [ ] **Roguelite 框架**: 实现 5 关循环与简单的 3选1 数据结构。
  * [ ] **美术 Pipeline**: 确定美术资源导入规范，美术开始产出白模/初版资源。

### Phase 2: 子系统与交互 (Week 3-4)
>
> **焦点**: 排兵布阵、拖拽交互、完整战斗

* **Week 3 (01/22 - 01/28)**: **编队系统周**
  * [ ] **排兵布阵**: 实现 3D 场景中的拖拽上阵功能 (Raycast, Grid Snapping)。
  * [ ] **数据绑定**: 编队数据与战斗初始化逻辑打通。
* **Week 4 (01/29 - 02/04)**: **战斗打磨周**
  * [ ] **表现层同步**: 完善受击反馈、飘字、血条 UI。
  * [ ] **Death Barrier**: 接入死亡屏障，确保游戏重开 (Restart) 逻辑亦无内存泄漏或状态污染。
  * [ ] **美术集成**: 替换所有白模为正式资源。

### Phase 3: 流程与引导 (Week 5-6)
>
> **焦点**: 完整游戏体验、新手引导

* **Week 5 (02/05 - 02/11)**: **Roguelite 循环周**
  * [ ] **3选1 UI**: 制作精美的 Buff 选择界面。
  * [ ] **结算流程**: 胜利/失败结算，数据重置/继承逻辑。
* **Week 6 (02/12 - 02/18)**: **引导与润色周**
  * [ ] **新手引导 (FTUE)**: 制作第一场战斗的强制引导（遮罩、手势）。
  * [ ] **视觉润色**: 增加场景特效（飘雪、雾气）、音效集成。

### Phase 4: 发布与交付 (Week 7-8)
>
> **焦点**: SDK、合规、真机测试

* **Week 7 (02/19 - 02/25)**: **平台对接周**
  * [ ] **TapTap SDK**: 接入（登录/时长统计/防沉迷壳子）。
  * [ ] **隐私合规**: 制作隐私弹窗，确保合规。
  * [ ] **CDN 部署**: 部署资源到真实 CDN 环境。
* **Week 8 (02/26 - 03/05)**: **QA 与上线周**
  * [ ] **真机测试**: 覆盖 Android 主流机型，测试刘海屏、性能 (GC/FPS)。
  * [ ] **Bug 修复**: 集中修 Bug。
  * [ ] **发布**: 打包 Release 版本，提交 TapTap 审核。

---

## ⚠️ 风险控制 (Risk Management)

1. **HybridCLR 兼容性**: 若在 Week 2 遇到严重真机 Crash 无法解决，Week 3 立即降级为纯 AOT 模式，优先保证 Demo 运行。
2. **美术进度拖延**: 若 Week 4 美术资源未完成，使用 Asset Store 免费资源顶替，保证逻辑不阻塞。
3. **编队交互复杂**: 若 3D 拖拽手感调优耗时过长，降级为 2D 列表点击上阵。
