## 一、问题概述
TapTap 隐私检测反馈问题如下：
1. 未明示收集 Android ID、IP、传感器信息（10101）。
2. 未明示 SDK 收集个人信息目的、方式、范围（10104）。
3. 首次启动提前申请存储、位置权限（20107）。

## 二、整改措施
### 1. 隐私政策内容补充（已完成）
- 明确披露收集的个人信息类型：Android ID、IP 地址、传感器信息、设备信息。
- 明确披露第三方 SDK 收集信息的目的、方式、范围。
- 明确仅在相关业务场景申请存储与位置信息权限。

### 2. SDK 收集信息披露（已完成）
隐私政策新增「第三方产品或服务如何获得您的个人信息」章节，列明以下 SDK：
- TapSDK：Android ID、设备信息（登录与渠道统计）。
- TapADN 广告组件：Android ID、IP、设备信息（广告投放与效果统计）。
- OkHttp/系统网络库：IP 与网络请求信息（网络通信）。
- Kotlin 协程支持库：任务调度运行信息。
- Unity3D 引擎：传感器信息（渲染与性能优化）。

### 3. 权限申请时机（待技术版本配合/按要求执行）
- 不在首次启动时申请存储或位置权限。
- 仅在功能触发场景（如存档/截图/资源缓存、地区化服务）申请。

## 三、权限清单与用途
- 存储空间：用于存档、截图与资源缓存。
- 位置信息：用于地区化服务（如选择最佳服务器节点/区域活动展示）。

## 四、数据收集说明
- 设备标识符（Android ID）、设备信息：用于账号安全、性能优化与服务稳定。
- IP 地址：用于网络通信与安全风控。
- 传感器信息：用于 Unity 引擎渲染与性能适配。

## 五、用户同意机制
- 首次启动展示隐私政策弹窗，用户同意后方可初始化第三方 SDK。
- 用户可在系统设置中关闭已授权权限。

## 六、结论
隐私政策已完成补充，覆盖 TapTap 反馈的 10101/10104/20107 问题；权限申请时机调整为业务场景触发，符合平台要求。
